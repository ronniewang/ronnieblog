Redis 的一些最佳实践

  1. Reids单实例大小不要超过 20G，超过 20G 的建议使用 Redis Cluster

  2. 禁止执行危险操作，对于一些比较危险的命令，比如 shutdown、flushdb、flushall、keys等，要做 rename，不要让客户端调用，如果需要跑这些命令，需要联系运维人员进行操作
